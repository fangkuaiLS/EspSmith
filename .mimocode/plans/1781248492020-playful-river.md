# EspSmith 兼容 CodeWhale 和 MiMo Code 方案

## 现状分析

EspSmith 当前 AI 助手**深度耦合 CodeWhale**，体现在以下 6 个维度：

| 维度 | 当前硬编码位置 | CodeWhale 具体实现 |
|------|-------------|-------------------|
| **二进制发现** | `ai_assistant.rs:2054-2255` `ensure_codewhale_ready()` | 搜索 `codewhale.exe` / `codewhale` |
| **CLI 调用** | `ai_assistant.rs:431-436` | `codewhale --model <model> exec --auto --output-format stream-json <prompt>` |
| **流式协议** | `ai_assistant.rs:541-906` 事件循环 | 解析 `content`/`tool_use`/`tool_result`/`session_capture`/`usage`/`metadata`/`done` 事件 |
| **Session 管理** | `ai_assistant.rs:438-440` | `--session-id` 参数 + `session_capture` 事件 |
| **环境变量** | `ai_assistant.rs:463-471` | `DEEPSEEK_API_KEY`, IPC pipe 地址 |
| **安装流程** | `ai_assistant.rs:2172-2227` `setup_codewhale()` | npm install codewhale 或内嵌二进制 |
| **设置界面** | `SettingsDialog.tsx:28-416` | CodeWhale 状态检测、安装按钮 |
| **Tauri 打包** | `tauri.conf.json:38-41` | 打包 `binaries/codewhale.exe` |
| **前端 API** | `api.ts:4` 注释、`chatStore.ts:374` | CodeWhale 特定的 delete_file/exec_shell 事件处理 |

## 兼容方案设计

### 核心思路：抽象 `AIBackend` trait

```
┌─────────────────────────────────────────┐
│           EspSmith Frontend             │
├─────────────────────────────────────────┤
│     ai_assistant.rs (Rust Backend)      │
│  ┌─────────────────────────────────┐    │
│  │   AIBackend trait (抽象层)       │    │
│  ├──────────┬──────────────────────┤    │
│  │ CodeWhale│    MiMo Code         │    │
│  │ Backend  │    Backend           │    │
│  └──────────┴──────────────────────┘    │
└─────────────────────────────────────────┘
```

### 方案 A：配置驱动切换（推荐，改动最小）

在不引入 trait 抽象的前提下，通过配置 + 分支逻辑实现双后端：

#### 1. 新增 AI Backend 类型枚举

**文件**: `ai_assistant.rs`

```rust
#[derive(Debug, Clone, Serialize, Deserialize, PartialEq, Eq)]
#[serde(rename_all = "lowercase")]
pub enum AIBackendType {
    Codewhale,
    MimoCode,
}

impl Default for AIBackendType {
    fn default() -> Self {
        Self::Codewhale
    }
}
```

#### 2. 扩展 AIConfig

**文件**: `ai_assistant.rs:74-94`

```rust
pub struct AIConfig {
    // ... 现有字段 ...
    #[serde(default)]
    pub ai_backend: AIBackendType,
}
```

#### 3. 二进制发现——统一搜索策略

**文件**: `ai_assistant.rs` — `ensure_codewhale_ready()` → 重命名为 `ensure_ai_backend_ready()`

| 后端 | 搜索路径优先级 |
|------|-------------|
| CodeWhale | 1. 内嵌 `binaries/codewhale.exe` → 2. 本地 `~/.local/share/espsmith/codewhale/` → 3. 系统 PATH `codewhale` |
| MiMo Code | 1. 内嵌 `binaries/mimo-code.exe` → 2. 本地 `~/.local/share/espsmith/mimo-code/` → 3. 系统 PATH `mimo-code` |

```rust
fn ensure_ai_backend_ready() -> Result<PathBuf, String> {
    let backend_type = AI_CLIENT.try_lock()
        .map(|c| c.config.ai_backend.clone())
        .unwrap_or_default();

    match backend_type {
        AIBackendType::Codewhale => ensure_codewhale_ready(),
        AIBackendType::MimoCode => ensure_mimo_code_ready(),
    }
}
```

#### 4. CLI 调用——参数适配

**文件**: `ai_assistant.rs` — `ai_send_message()` 中命令构建部分

| 参数 | CodeWhale | MiMo Code |
|------|-----------|-----------|
| 模型 | `--model deepseek-v4-flash` | `--model <model>` (待确认) |
| 执行模式 | `exec --auto --output-format stream-json` | `chat --stream-json` (待确认) |
| 会话 | `--session-id <id>` | `--conversation-id <id>` (待确认) |
| API Key | 环境变量 `DEEPSEEK_API_KEY` | 环境变量 `MIMO_API_KEY` (待确认) |

需要为 MiMo Code 实现对应的 `build_mimo_code_command()` 函数。

#### 5. 流式协议适配

**文件**: `ai_assistant.rs:530-928` 事件循环

两种后端的事件格式可能不同。核心事件映射：

| 功能 | CodeWhale 事件 | MiMo Code 事件 (待确认) |
|------|---------------|----------------------|
| 文本输出 | `{"type":"content","content":"..."}` | `{"type":"text","text":"..."}` |
| 工具调用 | `{"type":"tool_use","name":"...","input":{}}` | `{"type":"tool_call","function":"...","arguments":{}}` |
| 工具结果 | `{"type":"tool_result","tool_use_id":"...","content":"..."}` | `{"type":"tool_result","call_id":"...","output":"..."}` |
| 会话 ID | `{"type":"session_capture","content":"sid"}` | `{"type":"session","id":"..."}` |
| 用量统计 | `{"type":"usage","input_tokens":N}` | `{"type":"usage","prompt_tokens":N}` |
| 完成 | `{"type":"done","content":"..."}` | `{"type":"done"}` |

### 方案 B：Trait 抽象（更干净，改动较大）

定义 `AIBackend` trait，每种后端实现一个 struct：

```rust
#[async_trait]
trait AIBackend {
    fn name(&self) -> &str;
    fn binary_name(&self) -> &str;
    async fn ensure_ready(&self) -> Result<PathBuf, String>;
    fn build_command(&self, config: &AIConfig, prompt: &str) -> tokio::process::Command;
    fn parse_event(&self, line: &str) -> Option<AIStreamEvent>;
    fn env_vars(&self, config: &AIConfig) -> Vec<(String, String)>;
}

struct CodewhaleBackend;
struct MimoCodeBackend;
```

**优点**: 完全解耦，新增后端只需实现 trait
**缺点**: 改动量大，需要重构整个 ai_send_message 事件循环

---

## 推荐方案：方案 A（配置驱动）

理由：
1. 改动集中在 `ai_assistant.rs` 一个文件，风险可控
2. 不改变现有 CodeWhale 的任何行为
3. 前端只需加一个选择器
4. MiMo Code 接口确认后可快速适配

## 关键待确认项（需调研 MiMo Code）

| # | 问题 | 影响范围 |
|---|------|---------|
| 1 | MiMo Code CLI 的可执行文件名和安装方式 | 二进制发现、安装流程 |
| 2 | MiMo Code 的 CLI 参数格式（模型、流式输出、会话） | CLI 调用 |
| 3 | MiMo Code 的流式 JSON 事件格式 | 事件解析循环 |
| 4 | MiMo Code 使用的工具名（exec_shell 等）是否与 CodeWhale 一致 | 工具调用映射 |
| 5 | MiMo Code 的 API Key 环境变量名 | 环境变量注入 |
| 6 | MiMo Code 是否支持相同的 prompt 格式（AGENTS.md 等） | Prompt 构建 |

## 实施步骤

### Phase 1: 后端抽象 (ai_assistant.rs)
1. 新增 `AIBackendType` 枚举
2. 扩展 `AIConfig` 添加 `ai_backend` 字段
3. 重命名 `ensure_codewhale_ready()` → `ensure_ai_backend_ready()`
4. 实现 `ensure_mimo_code_ready()` 搜索逻辑
5. 实现 `build_mimo_code_command()` 命令构建
6. 在事件循环中根据 backend type 分支解析

### Phase 2: 前端适配
1. `settings.ts` 添加 `aiBackend` 字段
2. `SettingsDialog.tsx` 添加后端选择器
3. `api.ts` 在 `startAI()` 中传递 `aiBackend` 配置

### Phase 3: 打包与分发
1. `tauri.conf.json` 添加 mimo-code 资源
2. 考虑按需打包（用户只安装一个后端时减小包体积）

### Phase 4: 测试验证
1. 验证 CodeWhale 模式无回归
2. 验证 MiMo Code 模式端到端工作
3. 验证后端切换不需要重启应用
